# Plan 04.1-01 — Remove PredictIt end-to-end

## Objective

Remove PredictIt as a supported platform from the arbiter-dashboard codebase. Surgical cleanup driven by a prior Explore-agent audit. After completion, only Kalshi and Polymarket remain as supported venues. No architectural changes — just code subtraction and settings updates.

**User rationale (per user decision during Phase 4 verification):** PredictIt was assumed out earlier but never actually removed; its presence in the codebase is dead weight.

## Scope

Based on audit report (29 files, moderate blast radius):

1. **Delete** PredictIt-specific collector files (Python + TypeScript)
2. **Remove PredictIt references** from shared files (settings, main, engine, scanner, auditor, tests, UI, docs)
3. **Simplify platform-aware code** that previously dispatched across 3 platforms — now only 2 (Kalshi, Polymarket)
4. **Prune market-seed entries** — remove `"predictit": <id>` keys from MARKET_SEEDS dicts
5. **Update documentation** — CLAUDE.md, PROJECT.md

**Out of scope:**
- Migration of any existing PredictIt positions (there are none; system never went live)
- Renaming "cross-platform" → "dual-platform" language (cosmetic)
- TypeScript CLI consolidation beyond PredictIt removal

## Tasks

Delete:
- `arbiter/collectors/predictit.py`
- `arbiter/collectors/test_predictit_collector.py`
- `src/collectors/predictit-client.ts`

Verify nothing imports these (grep `from arbiter.collectors.predictit`, `predictit-client`).

## Risks

1. **Test-vector drift:** `test_math_auditor.py` and `src/__tests__/arbitrage-executor.test.ts` have constants tuned for 3-platform math. Removing predictit_cap or fee rows will fail some vectors. Fix by updating expected values, not by re-adding PredictIt.
2. **Dashboard filter UI:** If the Activity Atlas filter chip list is hardcoded with `["all", "kalshi", "polymarket", "predictit", "incident", "balance"]`, removing `"predictit"` shifts other indices. Grep dashboard.js for this array and edit.
3. **Market-seeds mapping:** Some canonical markets may have been mapped PredictIt-only. Removing their PredictIt key leaves them with only Kalshi or Polymarket, which is still valid (just single-venue — won't trigger arbitrage scanner but won't error).

## Not blocking

- Live-trading credential work (user is provisioning `.env` in parallel — that path is independent).
- Phase 4 verification status (Phase 4 stays in `human_needed` until operator runs live-fire; PredictIt removal doesn't change that).
