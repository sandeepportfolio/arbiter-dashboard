# Phase 04.1 Plan 01: Remove PredictIt end-to-end — Summary

PredictIt platform removed from the arbiter-dashboard codebase in a single surgical pass: 3 file deletions, 31 file edits spanning Python core, TypeScript CLI, web UI, SQL schema, and documentation. After this plan, only Kalshi and Polymarket remain as supported venues.

## Objective (achieved)

Delete PredictIt as a supported platform. No architectural changes — code subtraction and settings updates only.

## DB schema decision

SQL_INIT / migrate.py: both `predictit_id` and `predictit_contract_keywords` columns plus `idx_mappings_predictit` index were removed entirely (not kept dormant). Rationale: system has never gone live (DRY_RUN always true, zero `execution_history` rows in any deployment), so fresh deployments get the clean 2-platform schema. No production data migration needed.

## Deviations from plan

### Rule 1 — Bug fixed

**`arbiter/main.py::seed_dashboard_fixture`** — The plan only called out removing the PredictItCollector wiring, but the same function constructs PricePoints with `platform="predictit"` for `DEM_SENATE_2026` and `GOP_SENATE_2026`. Left as-is these would reintroduce dead platform data into the seed fixture. Swapped those PricePoints to `platform="kalshi"` with `fee_rate=0.07` and K-prefixed market ids so the dashboard smoke-seed still has 2-venue price pairs.

### Rule 2 — Missing critical functionality

**None.** Plan's Task list was complete after cross-referencing with the expanded scope from prior-agent's audit.

### Rule 3 — Blocking fixes

**`MARKET_SEEDS` `DEM_SENATE_2026` note text** — Note read "PredictIt only, so keep manual-only…" which would be misleading after removing the PredictIt id. Updated to "Polymarket only, so keep manual-only until a confirmed Kalshi leg exists." (functional equivalent; the auto-trade block still holds since it's still single-venue, and the note is now accurate).
