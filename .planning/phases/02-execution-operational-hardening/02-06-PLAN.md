Strip every platform-specific method from `arbiter/execution/engine.py` (the bodies are already cleanly extracted into `arbiter/execution/adapters/kalshi.py` and `polymarket.py` by Plans 04 and 05), inject `adapters: dict[str, PlatformAdapter]` and `store: Optional[ExecutionStore]` via the engine constructor, dispatch leg placement through `self.adapters[platform].place_fok(...)`, wrap each leg in `asyncio.wait_for(..., timeout=execution_timeout_s)` for EXEC-05, bind `arb_id`/`canonical_id` contextvars at the start of `execute_opportunity` for OPS-01 (Pitfall 6 mitigation), persist every order state transition + incident + arb-completion through the store, build `arbiter/execution/recovery.py` for restart reconciliation per D-17, and wire it all into `arbiter/main.py` so the system boots with the new architecture.

Purpose: Closes EXEC-04 (platform code fully relocated; engine is the platform-agnostic orchestrator), satisfies EXEC-05 (timeout + cancel-on-timeout), wires EXEC-02 persistence end-to-end (orders/fills/incidents/arbs persisted on every transition; restart-recovery hook attached at startup), and makes OPS-01 useful (contextvars binding produces correlated JSON log lines per arb).

       self,

       canonical_id: str,
       side: str,
       price: float,

               order_id=f"{arb_id}-{side.upper()}-NOADAPTER",

```python
"""Restart reconciliation hook for ExecutionEngine (EXEC-02 part 2 / D-17).

Pitfall 5 mitigation: a process crash mid-execution can leave the DB
showing orders in non-terminal status (pending/submitted/partial) when the
platform has actually settled them (filled/cancelled). Without reconciliation,
DB and platform drift -- in the worst case we have a ghost position.

This module is called from arbiter/main.py:run_system on startup, BEFORE
engine.run is called, so the engine begins with a coherent view of state.
"""
from __future__ import annotations

import logging
from typing import Dict, List

from .adapters.base import PlatformAdapter
from .engine import Order, OrderStatus
from .store import ExecutionStore

logger = logging.getLogger("arbiter.execution.recovery")


async def reconcile_non_terminal_orders(
    store: ExecutionStore,
    adapters: Dict[str, PlatformAdapter],
) -> List[Order]:
    """Reconcile DB-state vs platform-state for any non-terminal orders.

    Returns the list of orphaned orders (platform has no record of them).
    Caller (arbiter/main.py) should emit an incident for each orphaned order
    and let the operator decide whether to manually intervene.
    """
    orphaned: List[Order] = []
    try:
        orders = await store.list_non_terminal_orders()
    except Exception as exc:
        logger.error("recovery: failed to list non-terminal orders: %s", exc)
        return []

    if not orders:
        logger.info("recovery: no non-terminal orders to reconcile")
        return []

    logger.info("recovery: reconciling %d non-terminal orders", len(orders))

    for order in orders:
        adapter = adapters.get(order.platform)
        if adapter is None:
            logger.warning("recovery: no adapter for platform=%s order_id=%s",
                           order.platform, order.order_id)
            continue

        try:
            fresh = await adapter.get_order(order)
        except Exception as exc:
            logger.warning("recovery: get_order raised for %s: %s", order.order_id, exc)
            order.status = OrderStatus.FAILED
            order.error = f"orphaned on restart: {exc}"
            try:
                await store.upsert_order(order, arb_id=_derive_arb_id(order.order_id))
            except Exception as upsert_exc:
                logger.error("recovery: failed to mark orphaned %s: %s",
                             order.order_id, upsert_exc)
            orphaned.append(order)
            continue

        if fresh.status == OrderStatus.FAILED and "not found" in (fresh.error or "").lower():
            logger.info("recovery: orphaned (platform has no record) %s", order.order_id)
            try:
                await store.upsert_order(fresh, arb_id=_derive_arb_id(order.order_id))
            except Exception as upsert_exc:
                logger.error("recovery: failed to mark orphaned %s: %s",
                             order.order_id, upsert_exc)
            orphaned.append(fresh)
            continue

        if fresh.status != order.status:
            logger.info("recovery: reconciled %s old=%s new=%s",
                        order.order_id, order.status.value, fresh.status.value)
            try:
                await store.upsert_order(fresh, arb_id=_derive_arb_id(order.order_id))
            except Exception as upsert_exc:
                logger.error("recovery: failed to upsert reconciled %s: %s",
                             order.order_id, upsert_exc)

    logger.info("recovery: complete. orphaned=%d", len(orphaned))
    return orphaned


def _derive_arb_id(order_id: str) -> str:
    """ARB-NNNNNN-YES-... -> ARB-NNNNNN."""
    if not order_id or not order_id.startswith("ARB-"):
        return order_id
    parts = order_id.split("-")
    if len(parts) >= 2:
        return f"{parts[0]}-{parts[1]}"
    return order_id
```

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Engine -> Adapter (in-process) | Adapter contract is structural via Protocol; engine does NOT validate adapter return values further. |
| Engine -> ExecutionStore | persistence calls are wrapped in try/except so DB failures DEGRADE the engine (warning log) but do not BREAK execution. |
| Startup -> Reconcile -> Adapter | adapter.get_order may make external HTTP calls; recovery must not block startup indefinitely. |
| `arbiter/main.py` shared aiohttp session -> KalshiAdapter | one ClientSession is shared; cleanup must close it. |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-20 | DoS / Liveness | reconcile_non_terminal_orders blocking startup | mitigate | Per-order get_order calls are not parallelized; if orders list is large, startup may take seconds. For Phase 2 the volume is low (handful of orders); future work could use asyncio.gather with bounded concurrency. Catches all exceptions per-order so one stuck call cannot cascade. |
| T-02-21 | Tampering / silent data corruption | engine.store.upsert_order failure swallowed by try/except | mitigate | Failures logged at WARNING level and routed via Sentry's LoggingIntegration (Plan 01) — the operator sees them. The trade-off is that DB outages do NOT block execution (which would halt all trading), so we accept partial audit-trail loss on Postgres outage. Documented in PATTERNS.md design notes. |
| T-02-22 | Tampering (heartbeat lifecycle disruption) | adapter and heartbeat sharing ClobClient via factory | mitigate | Adapter receives `clob_client_factory=lambda: engine._get_poly_clob_client()` (Plan 05). Adapter NEVER calls post_heartbeat (Plan 05 acceptance criteria). main.py task list keeps `asyncio.create_task(engine.polymarket_heartbeat_loop(), name="poly-heartbeat")` (D-13 invariant test in acceptance criteria). |
| T-02-23 | Information Disclosure (orphaned-order incident may include PII) | engine.record_incident message constructed in main.py | mitigate | The message includes only `order.order_id`, `order.platform`, `order.error` — no private keys, no full order body. structlog `_strip_secrets` (Plan 01) provides defense-in-depth on any field name pattern. |
| T-02-24 | Spoofing / replay | reconcile re-runs upsert_order on already-terminal orders | accept | upsert_order with `ON CONFLICT (order_id) DO UPDATE` is idempotent; re-applying a FILLED status to an already-FILLED order is a no-op. recovery.py only writes when `fresh.status != order.status` (Task 2 implementation guard). |
