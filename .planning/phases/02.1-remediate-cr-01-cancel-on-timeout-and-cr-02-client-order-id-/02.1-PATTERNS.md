```python
except asyncio.TimeoutError:
    partial = Order(
        order_id=f"{arb_id}-{side.upper()}-{platform.upper()}",
        platform=platform, market_id=market_id, canonical_id=canonical_id,
        side=side, price=price, quantity=qty,
        status=OrderStatus.PENDING, timestamp=time.time(),
        error=f"local timeout after {self.execution_timeout_s}s",
    )
    cancelled_any = False
    prefix = f"{arb_id}-{side.upper()}-"
    try:
        open_orders = await adapter.list_open_orders_by_client_id(prefix)
    except Exception as exc:
        logger.warning(
            "timeout_recovery.lookup_failed platform=%s arb_id=%s err=%s",
            platform, arb_id, exc,
        )
        open_orders = []
    found_count = len(open_orders)
    for real in open_orders:
        try:
            if await adapter.cancel_order(real):
                cancelled_any = True
        except Exception as cancel_exc:
            logger.warning(
                "timeout_recovery.cancel_raised platform=%s order_id=%s err=%s",
                platform, real.order_id, cancel_exc,
            )
    if cancelled_any:
        partial.status = OrderStatus.CANCELLED
        partial.error += f"; cancelled {found_count} orphaned order(s)"
    elif found_count > 0:
        partial.status = OrderStatus.FAILED
        partial.error += (
            f"; found {found_count} orphaned order(s) but cancel failed"
            " - manual reconciliation required"
        )
    else:
        partial.status = OrderStatus.FAILED
        partial.error += (
            "; no matching open order found"
            " - platform may have rejected or never received"
        )
    order = partial
```

**Why this matches the file's conventions:**
- `partial.error += "; ..."` string-append pattern is already used at line 769.
- The event names (`timeout_recovery.lookup_failed`, `timeout_recovery.cancel_raised`) follow the dot-namespaced style seen in adapter structlog events (e.g. `kalshi.order.rejected`, `kalshi.cancel.failed`).
- No new `await` inside a lock, no new state on `self` — keeps the method stateless.

### 3. `arbiter/execution/engine.py` — `_derive_client_order_id` fix (CR-02)

**Analog:** Same file — the existing `@staticmethod _derive_client_order_id` at `engine.py:784-792` (the function being replaced) and its sibling `_derive_arb_id_from_order` at `engine.py:794-800`.

**Current buggy implementation** (lines 784-792):
```python
@staticmethod
def _derive_client_order_id(order: Order) -> Optional[str]:
    """Kalshi adapter uses ``{arb_id}-{SIDE}-{8-hex}`` as the client_order_id
    AND as the fallback order_id when the platform response lacks an id.
    For Polymarket there is no client_order_id concept; return None.
    """
    if order.platform == "kalshi" and order.order_id and "-" in order.order_id:
        return order.order_id
    return None
```

**Static-method + docstring style to preserve** (lines 784-800):
- `@staticmethod` decorator on both `_derive_client_order_id` and `_derive_arb_id_from_order`.
- Triple-quoted docstring with one-paragraph summary.
- Return type `Optional[str]` for "may not be derivable" semantics.

**Replacement pattern:**
```python
@staticmethod
def _derive_client_order_id(order: Order) -> Optional[str]:
    """Return the adapter-populated client_order_id, or None.

    Kalshi adapter populates ``Order.external_client_order_id`` with the
    ``ARB-{n}-{SIDE}-{hex}`` string used as the Kalshi idempotency key.
    Polymarket has no client_order_id concept and leaves it None.
    """
    return order.external_client_order_id
```

**Why this matches the file's conventions:** One-line body matches the concise style of nearby helpers; docstring explicitly cross-references the adapter's behavior (mirror of the pre-fix docstring's adapter reference).

## Shared Patterns

### Pattern A: `Order` dataclass as the adapter/engine seam DTO

**Source:** `arbiter/execution/engine.py:43-88` (`Order` dataclass + `to_dict` + `to_audit_dict`).

**Apply to:** All three production files (engine.py, kalshi.py, polymarket.py) plus all three test files.

**Invariants to preserve:**
1. `Order` is imported by both adapters from `..engine` (kalshi.py:21; polymarket.py has similar). Do NOT create a duplicate `Order` in adapters.
2. New fields go at the END of the dataclass after existing optional fields, with explicit `Optional[T] = None` defaults, preserving positional-construction back-compat.
3. `to_dict` and `to_audit_dict` should both be extended when a new field is added, to keep observability consistent (dashboard `to_dict` + audit log `to_audit_dict`).
4. Any field that represents "adapter-internal state the engine needs to see" lives on Order (current fields `error`, `fill_price`, `fill_qty` all fit this category).

---

### Pattern B: Engine-side error handling — never raise across `_place_order_for_leg`

**Source:** Existing `try/except Exception` wrappers in `engine.py:756-764` (the cancel_order wrapper) and `engine.py:780-781` (the store upsert wrapper).

**Apply to:** Both new engine calls in the rewritten timeout branch (`list_open_orders_by_client_id` + each per-order `cancel_order`).

**Excerpt** (lines 780-781):
```python
try:
    client_order_id = self._derive_client_order_id(order)
    await self.store.upsert_order(order, arb_id=arb_id, client_order_id=client_order_id)
except Exception as exc:
    logger.warning("ExecutionStore upsert_order failed: %s", exc)
```

**Why this matters for CR-01:** The timeout branch's new `list_open_orders_by_client_id` call CAN fail (network error, Kalshi 5xx, rate limit). The engine must log-and-continue, not re-raise — the caller `_live_execution` expects `_place_order_for_leg` to always return an Order. The reference implementation in RESEARCH.md Example 1 preserves this by wrapping each adapter call in its own `try/except`.

### Pattern C: Structlog vs stdlib logging split

**Source:** `engine.py:29` uses stdlib `logger = logging.getLogger("arbiter.execution")`; adapters at `kalshi.py:24` and `polymarket.py:~20` use `log = structlog.get_logger("arbiter.adapters.*")`.

**Apply to:**
- New engine-side log calls (timeout_recovery.*) → stdlib `logger.warning("event_name key=%s ...", val, ...)`.
