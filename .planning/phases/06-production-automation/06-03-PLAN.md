<name>Task 1: TelegramNotifier with send/retry/dedup</name>
<files>arbiter/notifiers/telegram.py, arbiter/notifiers/test_telegram.py</files>
<action>
Create arbiter/notifiers/telegram.py exporting `TelegramNotifier`:
  - `__init__(bot_token, chat_id, rate_limit_sec=60)` — pass None for disabled mode (no-op).
  - `async send(event_type: str, message: str, dedup_key: str | None = None) -> bool` — posts to https://api.telegram.org/bot{token}/sendMessage with chat_id+text. Returns True on 200, False on any error.
  - Retries on aiohttp.ClientError with exponential backoff (0.5s, 1s, 2s), max 3 attempts.
  - Dedup: keeps a dict `{dedup_key: timestamp}` of last-sent per key; skips if within rate_limit_sec window; logs `telegram.deduped`.
  - Disabled mode (token or chat_id is None/empty): returns True immediately without network call, logs `telegram.disabled`.
  - All errors logged; no exceptions propagate.

RED tests (AsyncMock for aiohttp session):
  - test_disabled_returns_true_no_network: no token → True, no HTTP call
  - test_success_200: returns True on 200
  - test_retry_on_transient: 3rd attempt succeeds
  - test_retry_exhausted: 3 failures → False
  - test_dedup_within_window: same key within 60s → second call skipped
  - test_dedup_different_keys: different keys always sent
  - test_dedup_expired_window: same key after 60s → resent
</action>
</task>
