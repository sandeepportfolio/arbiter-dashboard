<objective>
Wire `ArbitrageScanner` → `ExecutionEngine` so that when the scanner emits an `ArbitrageOpportunity` that is (a) persisted past `persistence_scans`, (b) has `allow_auto_trade=True` on its `MarketMapping`, (c) is below `MAX_POSITION_USD`, AND (d) the `SafetySupervisor` is NOT armed, the engine executes the opportunity automatically — no operator approval per trade.

Bootstrap rollout: `PHASE5_BOOTSTRAP_TRADES` env var caps the number of auto-executions to N for the initial rollout. Once cleared, remove the cap.
</objective>

<tasks>

<task type="auto">
<name>Task 1: add AutoExecutor wiring in arbiter.main</name>
<files>arbiter/main.py, arbiter/execution/engine.py</files>
<action>
Add an async `AutoExecutor` wiring function in arbiter/main.py that:
  1. Subscribes to `scanner.subscribe(callback)` with an async callback `_on_opportunity`.
  2. The callback:
     a. Skips if `safety_supervisor.is_armed` → log `auto_execute.skip.armed`.
     b. Skips if opportunity.mapping.allow_auto_trade is False → log `auto_execute.skip.not_allowed`.
     c. Skips if opportunity.total_notional_usd > MAX_POSITION_USD → log `auto_execute.skip.over_cap`.
     d. Skips if `PHASE5_BOOTSTRAP_TRADES` is set AND counter >= budget → log `auto_execute.skip.bootstrap_cap_reached`.
     e. Else: `await engine.execute(opportunity)`. Increments bootstrap counter on successful enter.
     f. Fire-and-forget; exceptions caught + logged with `auto_execute.error`, never bubble.
  3. Idempotent: tracks `seen_opportunity_ids` set (canonical_id + timestamp rounded to second) to avoid double-fire on scanner re-emit.

Wire it in `run_system()` after scanner/engine are constructed, before `await shutdown_event.wait()`.
</action>
