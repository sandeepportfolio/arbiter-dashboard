Section 1 — "Prereqs": docker + docker-compose v2, node 18+, python 3.12, git, a browser (for Kalshi + Polymarket portals).

Section 2 — "Credentials you must provision yourself (I cannot automate these):":
  A. **Kalshi production API key**:
     - Open https://kalshi.com (NOT demo-api). Log in.
     - Navigate: Account → API Keys → Create Key. Copy the Key ID.
     - Download the RSA private key (it's only shown once). Save to `./keys/kalshi_private.pem` (this path is gitignored).
     - Fund the Kalshi account with ≥$100 via ACH/wire (Account → Deposit).
  B. **Polymarket wallet**:
     - Generate a FRESH Polygon wallet (MetaMask → Create Account; use a NEW seed phrase for the arbiter, not a personal wallet).
     - Export the private key (MetaMask → Account Details → Show Private Key). Save it — you'll paste it into .env.production. NEVER commit it.
     - Note the wallet address (this is POLY_FUNDER).
     - Deposit 20–50 USDC to the wallet on Polygon (via Coinbase → Send → Polygon network, or any USDC bridge).
     - Open https://polymarket.com, connect the wallet, deposit USDC into Polymarket's trading balance (UI flow).
  C. **Telegram bot**:
     - Message `@BotFather` on Telegram → `/newbot` → follow prompts → receive BOT_TOKEN.
     - Message `@userinfobot` → receive your chat id (CHAT_ID).

Section 4 — "Bring up production stack":
```bash
docker compose -f docker-compose.prod.yml up -d
curl http://localhost:8080/api/health   # expect {"status":"ok"}
```

Section 5 — "Telegram dry-test":
```bash
docker compose -f docker-compose.prod.yml exec arbiter-api-prod python -m arbiter.notifiers.telegram
```
Expect Telegram message: "Arbiter Telegram dry-test OK ✓"

Section 6 — "Preflight":
```bash
docker compose -f docker-compose.prod.yml exec arbiter-api-prod python -m arbiter.live.preflight
```
All 15 items must pass. Fix any blocker before proceeding.

Section 7 — "Curate MARKET_MAP":
- Open http://localhost:8080/ops → log in with OPS_EMAIL/OPS_PASSWORD.
- Navigate to Mappings panel.
- Pick one Kalshi↔Polymarket pair that has identical-meaning resolution. Toggle `allow_auto_trade=true`.

Section 8 — "First supervised trade (Phase 5 Plan 05-02 Task 3b)":
```bash
docker compose -f docker-compose.prod.yml exec arbiter-api-prod \
  pytest -m live --live arbiter/live/test_first_live_trade.py -v -s
```
Watch the dashboard. You have 60 seconds to hit kill-switch after pre_trade_requote.json is written. Test places ONE trade, waits for settlement, reconciles. If reconcile PASS → phase_gate_status flips to PASS. If reconcile breach → auto_abort trips kill-switch — this is also a pass (proves the safety path).

Section 9 — "Flip to auto-mode":
- Once Section 8 passes cleanly: edit .env.production → `AUTO_EXECUTE_ENABLED=true`.
- Remove the `PHASE5_BOOTSTRAP_TRADES=5` cap once you're confident.
- Restart: `docker compose -f docker-compose.prod.yml restart arbiter-api-prod`.
- Watch /api/metrics + /ops dashboard. System now auto-trades any allow_auto_trade=true market pairs.

Section 10 — "Rollback / shutdown":
- Kill-switch (instant, cancels all open orders): `curl -X POST http://localhost:8080/api/kill-switch -d '{"action":"arm","reason":"manual"}' -H "Cookie: arbiter_session=<token>"`
- Graceful shutdown (SIGTERM): `docker compose -f docker-compose.prod.yml down`

Section 11 — "Troubleshooting matrix":
| Symptom | Likely cause | Fix |
|---------|--------------|-----|
| /api/readiness `kalshi_authenticated: false` | Key missing or wrong path | Check keys/kalshi_private.pem exists + readable |
| Preflight #10 fails | Telegram not configured | Run `python -m arbiter.notifiers.telegram` |
| AutoExecutor never fires | `AUTO_EXECUTE_ENABLED=false` or no mapping has `allow_auto_trade=true` | Flip env + toggle mapping in dashboard |
| Kill-switch stuck ARMED | Cooldown active | Wait cooldown_remaining seconds, then /api/kill-switch reset |
| Orders stuck SUBMITTED after restart | Crash recovery | arbiter.main calls reconcile_non_terminal_orders on startup; check logs |
