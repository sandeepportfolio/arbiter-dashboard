<objective>
Produce the production deployment bundle so the system runs 24×7 with auto-restart, log rotation, and crash-recovery reconciliation on startup.
</objective>

<tasks>

<task type="auto">
<name>Task 1: docker-compose.prod.yml</name>
<files>docker-compose.prod.yml</files>
<action>
Create docker-compose.prod.yml with three services:
  1. `arbiter-postgres-prod`: postgres:16-alpine, POSTGRES_USER/PASSWORD/DB env vars from .env.production, healthcheck (pg_isready), restart=always, named volume `arbiter_postgres_prod_data`, port NOT exposed to host (only accessible within the compose network).
  2. `arbiter-redis-prod`: redis:7-alpine, restart=always, named volume, healthcheck.
  3. `arbiter-api-prod`: builds from ./Dockerfile, depends_on healthchecks, restart=always, env_file: .env.production, reads keys/*.pem from mounted volume (./keys:/app/keys:ro), exposes 8080 to host, healthcheck via /api/health.

Log to stdout (docker default) with json-file driver, max-size=50m, max-file=5.

Add network: arbiter_prod_net.
</action>
<verify>
<automated>docker compose -f docker-compose.prod.yml config</automated>
</verify>
</task>

<task type="auto">
<name>Task 2: systemd unit template for bare-metal operators</name>
<files>deploy/systemd/arbiter.service</files>
<action>
Create deploy/systemd/arbiter.service that runs `python -m arbiter.main` under a dedicated `arbiter` system user, EnvironmentFile=/etc/arbiter/arbiter.env, Restart=always, RestartSec=5s, KillSignal=SIGTERM (matches existing shutdown handler), TimeoutStopSec=30s.
</action>
</task>

<task type="auto">
<name>Task 3: logrotate config</name>
<files>deploy/logrotate/arbiter.conf</files>
<action>
Create deploy/logrotate/arbiter.conf for /var/log/arbiter/*.log: daily, rotate 14, compress, delaycompress, missingok, notifempty, copytruncate.
</action>
</task>

<task type="auto">
<name>Task 4: startup reconcile hook in arbiter.main</name>
<files>arbiter/main.py</files>
<action>
Before starting the scanner, invoke `arbiter.execution.recovery.reconcile_non_terminal_orders(engine)` to reconcile any orders left in non-terminal states by a prior crash. This already exists per Phase 2.1 — just call it explicitly on startup and log the result.
</action>
</task>

<task type="auto">
<name>Task 5: deploy/README.md</name>
<files>deploy/README.md</files>
<action>
Operator runbook:
  - Prereqs: docker + docker compose v2 (or systemd+python3.12 for bare-metal)
  - Bootstrap: cp .env.production.template .env.production; fill in secrets
  - docker path: `docker compose -f docker-compose.prod.yml up -d`
  - systemd path: `sudo cp deploy/systemd/arbiter.service /etc/systemd/system/ && systemctl enable --now arbiter`
  - Health check: `curl http://localhost:8080/api/health`
  - Logs: `docker compose -f docker-compose.prod.yml logs -f arbiter-api-prod` OR `journalctl -u arbiter -f`
  - Shutdown: `docker compose down` OR `systemctl stop arbiter` (both trigger the SAFE-05 graceful-shutdown sequence)
</action>
</task>

</tasks>
