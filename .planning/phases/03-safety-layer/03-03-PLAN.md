<objective>
Promote `_recover_one_leg_risk` from "log a generic incident" to "log a structured event + Telegram alert + dedicated WebSocket event + unwind recommendation". This makes a naked position impossible to miss: it appears in the incident queue (existing), fires a high-visibility Telegram alert, and flows as a dedicated `one_leg_exposure` WebSocket event so plan 03-07 can build a hero-level UI banner.

Purpose: SAFE-03 is the most time-sensitive failure mode — a naked position can move against the operator minute-by-minute. Current code emits a "critical" incident but it's lost among other incidents. This plan adds three independent notification channels (incident queue, Telegram, dedicated WS event) so at least one always reaches the operator.

| T-3-03-F | Tampering | Bot token leaked via log file containing alert body | mitigate | Existing TelegramNotifier logs truncated previews only (`{message[:80]}...`); token is never logged (held in self._token). |
