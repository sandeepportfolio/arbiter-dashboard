<objective>
Build the SafetySupervisor module that owns kill-switch state, gates ExecutionEngine at every opportunity, cancels orders across all adapters in parallel on arm, persists audit events (Redis live + Postgres audit), and publishes state over WebSocket. Wire POST /api/kill-switch (arm/reset) endpoints with operator auth, and ensure the dashboard JS tolerates the new kill_switch WebSocket event type without breaking any existing panel.

Purpose: SAFE-01 is the phase's foundation — every other safety mechanism (rate limits, one-leg, shutdown) piggy-backs on the supervisor's subscribe/publish infrastructure and allow_execution gate. This plan delivers the core plumbing plus a minimal, non-breaking dashboard hook so later plans can add UI on top of a working event stream.

- New `arbiter/safety/` package (4 modules + 3 test files)
